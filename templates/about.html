El Atlas de Raíces Arameas es un explorador de raíces trilíteras que abarca unos 1.500 años de historia literaria aramea. Reúne cuatro corpus — la Peshitta (NT y AT), el arameo bíblico y el Tárgum Onkelos — bajo un único índice de raíces consonantícas, permitiendo rastrear una misma raíz semítica a través de tradiciones textuales, escrituras y siglos, con cognados hebreos y árabes visibles en cada capa.

{% if lang == 'es' %}Metodología{% elif lang == 'he' %}מתודולוגיה{% elif lang == 'ar' %}المنهجية{% else %}Methodology{% endif %}

{% if lang == 'es' %}

Extracción de raíces

El motor extrae raíces trilíteras consonantícas a partir de texto no vocalizado mediante análisis morfológico: eliminación sistemática de prefijos y sufijos, puntuación ponderada contra un diccionario de raíces verificadas, y resolución de candidatos ambiguos por frecuencia y contexto.

Normalización entre escrituras

Los textos arameos emplean dos sistemas de escritura: siríaco (ܐ–ܬ) y hebreo cuadrado (א–ת). El Atlas normaliza ambos a una clave latina común — por ejemplo, el siríaco ܟܬܒ y el hebreo כתב se resuelven a K-T-B — lo que permite la búsqueda unificada de raíces a través de los corpus.

Análisis morfológico

Se aplican conjuntos separados de reglas de afijos para el siríaco y para el arameo bíblico en escritura hebrea, dando cuenta de las diferencias en preposiciones clíticas, marcadores pronominales y morfología verbal entre ambas tradiciones.

Puntuación de confianza

Cada atribución de raíz lleva una puntuación de confianza (0.0–1.0) que refleja cómo fue identificada la raíz. Haga clic en cualquier palabra del lector para ver su puntuación como un indicador de color:

  • • Alta (≥ 0,8) — Palabra encontrada en el diccionario verificado, o triliteral pura que coincide con una raíz conocida. Estas atribuciones son fiables.
  • • Media (0,5–0,8) — Raíz extraída mediante eliminación de afijos pero no verificada independientemente contra el diccionario. Plausible pero debe verificarse en casos límite.
  • • Baja (< 0,5) — Raíz reconstruida a partir de expansión de letras débiles o eliminación morfológica intensa. Estas son las más propensas a errores.

La puntuación es estadística, no basada en etiquetado morfológico. No indica la forma morfológica (Peal, Aphel, Ethpeel), el patrón de la raíz ni la función gramatical de la palabra.

{% elif lang == 'he' %}

חילוץ שורשים

המנוע מחלץ שורשים תלת-עיצוריים מטקסט בלתי מנוקד באמצעות ניתוח מורפולוגי: הפשטה שיטתית של תחיליות וסופיות, ניקוד משוקלל כנגד מילון שורשים מאומתים, ופתרון מועמדים דו-משמעיים לפי תדירות והקשר.

נרמול בין כתבים

הטקסטים הארמיים משתמשים בשני כתבים: סורי (ܐ–ܬ) ועברי מרובע (א–ת). האטלס מנרמל את שניהם למפתח לטיני משותף — למשל, הסורי ܟܬܒ והעברי כתב נפתרים ל-K-T-B — מה שמאפשר חיפוש שורשים מאוחד בין הקורפוסים.

ניתוח מורפולוגי

מערכות חוקי ניתוח נפרדות מוחלות על הסורית ועל הארמית המקראית בכתב העברי, תוך התייחסות להבדלים במילות יחס קליטיות, סמני כינויים ומורפולוגיה פועלית בין שתי המסורות.

ניקוד ביטחון

כל ייחוס שורש נושא ציון ביטחון (0.0–1.0) המשקף כיצד זוהה השורש. לחצו על כל מילה בקורא כדי לראות את הציון כמחוון צבעוני:

  • • גבוה (≥ 0.8) — מילה שנמצאה במילון המאומת, או תלת-עיצורית חשופה התואמת שורש ידוע. ייחוסים אלה אמינים.
  • • בינוני (0.5–0.8) — שורש שחולץ באמצעות הפשטת מוספיות אך לא אומת באופן עצמאי מול המילון. סביר אך יש לבדוק במקרי קצה.
  • • נמוך (< 0.5) — שורש ששוחזר מהרחבת אותיות חלשות או הפשטה מורפולוגית כבדה. אלה הסבירים ביותר להכיל שגיאות.

הניקוד סטטיסטי, לא מבוסס על תיוג מורפולוגי. הוא אינו מציין את הצורה המורפולוגית (פעל, אפעל, אתפעל), את תבנית הגזע או את התפקיד הדקדוקי של המילה.

{% elif lang == 'ar' %}

استخلاص الجذور

يستخلص المحرّك الجذور الثلاثية الصوامتية من النص غير المُشكَل عبر التحليل الصرفي: تجريد منهجي للسوابق واللواحق، وتسجيل مرجّح مقابل معجم جذور موثّقة، وحلّ المرشّحين المتعدّدين بحسب التواتر والسياق.

التطبيع بين الخطوط

تستعمل النصوص الآرامية خطّين كتابيين: السرياني (ܐ–ܬ) والعبري المربع (א–ת). يُطبّع الأطلس كليهما إلى مفتاح لاتيني مشترك — فمثلاً السرياني ܟܬܒ والعبري כתב يُحلّان إلى K-T-B — ممّا يتيح البحث الموحّد عن الجذور عبر المدوّنات.

التحليل الصرفي

تُطبَّق مجموعات منفصلة من قواعد الزوائد على السريانية وعلى الآرامية التوراتية بالخط العبري، مع مراعاة الاختلافات في حروف الجر المتّصلة وعلامات الضمائر والصرف الفعلي بين التقليدين.

تسجيل الثقة

يحمل كل نسب جذري درجة ثقة (0.0–1.0) تعكس كيفية تحديد الجذر. انقر على أي كلمة في القارئ لرؤية درجتها كمؤشر لوني:

  • • عالية (≥ 0.8) — كلمة موجودة في المعجم الموثّق، أو ثلاثية صوامت مطابقة لجذر معروف. هذه النسب موثوقة.
  • • متوسطة (0.5–0.8) — جذر مُستخلَص عبر تجريد الزوائد لكنه لم يُتحقَّق منه مستقلاً في المعجم. محتمل لكن يجب التحقق في الحالات الحدّية.
  • • منخفضة (< 0.5) — جذر أُعيد بناؤه من توسيع حروف العلة أو تجريد صرفي مكثف. هذه الأكثر عرضة للخطأ.

التسجيل إحصائي وليس مبنياً على الوسم الصرفي. لا يُشير إلى الوزن الصرفي (فعل، أفعل، إتفعل)، ولا إلى نمط الجذع أو الوظيفة النحوية للكلمة.

Root Extraction

The engine extracts triliteral consonantal roots from unvocalized text through morphological analysis: systematic prefix/suffix stripping, weighted scoring against a dictionary of verified roots, and resolution of ambiguous candidates by frequency and context.

Cross-Script Normalization

The Aramaic texts employ two writing systems: Syriac (ܐ–ܬ) and Hebrew square script (א–ת). The Atlas normalizes both to a shared Latin key — for example, Syriac ܟܬܒ and Hebrew כתב both resolve to K-T-B — enabling unified root search across corpora.

Morphological Analysis

Separate affix rule sets are applied for Syriac and for Biblical Aramaic in Hebrew script, accounting for differences in clitic prepositions, pronominal markers, and verbal morphology between the two traditions.

Root Confidence Scoring

Each root attribution carries a confidence score (0.0–1.0) reflecting how the root was identified. Click any word in the reader to see its score as a colored indicator:

  • • High (≥ 0.8) — Word found in the verified dictionary, or a bare triliteral matching a known root. These attributions are reliable.
  • • Medium (0.5–0.8) — Root extracted via affix stripping but not independently verified against the dictionary. Plausible but should be checked for edge cases.
  • • Low (< 0.5) — Root reconstructed from weak-letter expansion or heavy morphological stripping. These are the most likely to contain errors, particularly for weak roots (I-ʾAlap, II-Waw/Yod, III-ʾAlap) and quadriliteral forms.

The scoring is statistical, not based on morphological tagging. It does not indicate the morphological form (Peal, Aphel, Ethpeel), stem pattern, or grammatical function of the word. No error rate has been measured against a gold-standard morphological corpus.

{% if lang == 'es' %}Cognados y estructura semántica{% elif lang == 'he' %}קוגנטים ומבנה סמנטי{% elif lang == 'ar' %}المُشتركات والبنية الدلالية{% else %}Cognates & Semantic Structure{% endif %}

{% if lang == 'es' %}

Los cognados son palabras en lenguas semíticas emparentadas que comparten un ancestro radical común. Por ejemplo, el siríaco ܫܠܡܐ (shlama), el hebreo שָׁלוֹם (shalom) y el árabe سَلام (salām) derivan de la raíz SH-L-M, cuyo núcleo semántico gira en torno a «paz / plenitud / completud».

El Atlas contiene 1.127 familias de cognados. Cada entrada incluye: clave de raíz, glosas en inglés y español, palabras cognadas en hebreo y árabe con transliteración y significado, puentes semánticos entre cognados cuyos significados divergieron, marcadores de outlier para cognados con desplazamiento semántico significativo, raíces hermanas que comparten 2 de 3 consonantes, y el sabor de raíz — una línea poética que captura la intuición semítica del esqueleto consonantíco.

De las 1.127 entradas, 493 fueron generadas con asistencia de inteligencia artificial (API de Claude, Anthropic) a partir de listas de raíces, y revisadas y curadas manualmente para garantizar la precisión lingüística.

{% elif lang == 'he' %}

קוגנטים הם מילים בשפות שמיות קרובות החולקות שורש אב משותף. למשל, הסורי ܫܠܡܐ (shlama), העברי שָׁלוֹם (shalom) והערבי سَلام (salām) נגזרים מהשורש SH-L-M, שגרעינו הסמנטי סובב סביב «שלום / שלמות / השלמה».

האטלס מכיל 1,127 משפחות קוגנטים. כל רשומה כוללת: מפתח שורש, פירושים באנגלית ובספרדית, מילים קוגנטיות בעברית ובערבית עם תעתיק ומשמעות, גשרים סמנטיים בין קוגנטים שמשמעותם התפצלה, סמני חריג לקוגנטים עם הסטה סמנטית משמעותית, שורשים אחיים החולקים 2 מתוך 3 עיצורים, וטעם השורש — שורה פואטית הלוכדת את האינטואיציה השמית של שלד העיצורים.

מתוך 1,127 הרשומות, 493 נוצרו בסיוע בינה מלאכותית (API של Claude, Anthropic) מרשימות שורשים, ונבדקו ונערכו ידנית להבטחת דיוק לשוני.

{% elif lang == 'ar' %}

المُشتركات (cognates) هي كلمات في لغات سامية مترابطة تتشارك جذراً أصلياً واحداً. فمثلاً، السريانية ܫܠܡܐ (shlama)، والعبرية שָׁלוֹם (shalom)، والعربية سَلام (salām) تشتقّ جميعها من الجذر SH-L-M، الذي تدور نواته الدلالية حول «السلام / الكمال / التمام».

يحتوي الأطلس على 1,127 عائلة اشتقاقية. تشمل كل مدخلة: مفتاح الجذر، ترجمات بالإنجليزية والإسبانية، كلمات مشتركة بالعبرية والعربية مع الكتابة الصوتية والمعنى، جسوراً دلالية بين المشتركات التي تباعدت معانيها، علامات شذوذ للمشتركات ذات الانزياح الدلالي الكبير، جذوراً شقيقة تتشارك حرفين من ثلاثة، ونكهة الجذر — سطر شعري يلتقط الحدس السامي للهيكل الصوامتي.

من أصل 1,127 مدخلة، وُلّدت 493 منها بمساعدة الذكاء الاصطناعي (واجهة Claude، Anthropic) انطلاقاً من قوائم جذور، ثم رُوجعت ونُقّحت يدوياً لضمان الدقة اللغوية.

{% else %}

Cognates are words in related Semitic languages that share a common root ancestor. For example, Syriac ܫܠܡܐ (shlama), Hebrew שָׁלוֹם (shalom), and Arabic سَلام (salām) all derive from the root SH-L-M, whose semantic core revolves around "peace / wholeness / completion."

The Atlas contains 1,127 cognate families. Each entry includes: root key, English and Spanish glosses, Hebrew and Arabic cognate words with transliteration and meaning, semantic bridges linking cognates whose meanings diverged, outlier flags for cognates with significant semantic drift, sister roots sharing 2 of 3 consonants, and the root flavor (sabor de raíz) — a poetic one-liner capturing the Semitic intuition behind the consonantal skeleton.

Of the 1,127 entries, 493 were generated with AI assistance (Claude API, Anthropic) from root lists, then manually reviewed and curated for linguistic accuracy.

{# ── 6b. METHODOLOGICAL NOTES ── #}

{% if lang == 'es' %}Notas metodológicas{% elif lang == 'he' %}הערות מתודולוגיות{% elif lang == 'ar' %}ملاحظات منهجية{% else %}Methodological Notes{% endif %}

{% if lang == 'es' %}

La Peshitta como traducción. La posición académica mayoritaria sostiene que el Nuevo Testamento de la Peshitta es en gran medida una traducción del griego, no una composición aramea independiente. El análisis de raíces refleja las opciones léxicas del traductor, no necesariamente el vocabulario original del autor. El Antiguo Testamento de la Peshitta fue traducido principalmente del hebreo, aunque algunas porciones pueden preservar tradiciones arameas independientes.

Los tárgumes como traducciones interpretativas. El Tárgum Onkelos es una interpretación aramea del Pentateuco hebreo. Su vocabulario refleja la paráfrasis y expansión exegética del targumista, no una correspondencia directa con el texto hebreo fuente.

Limitaciones de la extracción de raíces. La extracción de raíces se realiza estadísticamente mediante eliminación de afijos y cotejo con diccionario, sin etiquetado morfológico ni anotación de categorías gramaticales. No se ha medido una tasa formal de error. Las raíces con letras débiles (’alep, waw, yod) y las formas cuadrilitéras son particularmente propensas a identificación errónea. Los usuarios deben tratar las atribuciones de raíces como probabilísticas, no definitivas.

Cognados generados con IA. De las 1.127 entradas de cognados, 493 fueron generadas con asistencia de IA (Claude API) y revisadas manualmente. Los investigadores que realicen trabajo formal deben verificar las relaciones de cognados de manera independiente.

{% elif lang == 'he' %}

הפשיטתא כתרגום. עמדת המחקר הרווחת גורסת שהברית החדשה בפשיטתא היא ברובה תרגום מיוונית, ולא חיבור ארמי עצמאי. ניתוח השורשים משקף את בחירות המתרגם, לא בהכרח את אוצר המילים המקורי של המחבר. התנ"ך בפשיטתא תורגם בעיקר מעברית, אם כי חלקים מסוימים עשויים לשמר מסורות ארמיות עצמאיות.

תרגומים כתרגומים פרשניים. תרגום אונקלוס הוא תרגום ארמי פרשני של התורה העברית. אוצר המילים שלו משקף פרפרזה והרחבה פרשנית של המתרגם, לא התאמה ישירה לטקסט העברי המקורי.

מגבלות חילוץ השורשים. חילוץ השורשים מתבצע סטטיסטית באמצעות הסרת תחיליות וסיומות והתאמה למילון, ללא תיוג מורפולוגי או סיווג חלקי דיבר. לא נמדד שיעור שגיאות פורמלי. שורשים עם אותיות חלשות (אלף, וו, יוד) וצורות מרובעות עלולים להיות מזוהים באופן שגוי. יש להתייחס לייחוסי השורשים כהסתברותיים, לא כוודאיים.

קוגנטים שנוצרו בסיוע AI. מתוך 1,127 ערכי הקוגנטים, 493 נוצרו בסיוע בינה מלאכותית (Claude API) ונבדקו ידנית. חוקרים המבצעים עבודה פורמלית מתבקשים לאמת את קשרי הקוגנטים באופן עצמאי.

{% elif lang == 'ar' %}

الپشيطتا كترجمة. يرى الموقف الأكاديمي السائد أن العهد الجديد في الپشيطتا هو إلى حدٍّ كبير ترجمة عن اليونانية، وليس تأليفاً آرامياً مستقلاً. تحليل الجذور يعكس خيارات المترجم المعجمية، لا بالضرورة مفردات المؤلف الأصلي. أما العهد القديم في الپشيطتا فقد تُرجم أساساً عن العبرية، وإن كانت بعض الأجزاء قد تحفظ تقاليد آرامية مستقلة.

الترجومات كترجمات تفسيرية. ترجوم أونكيلوس هو ترجمة آرامية تفسيرية للتوراة العبرية. مفرداته تعكس صياغة المترجم وتوسّعه التفسيري، لا مطابقةً حرفية للنص العبري المصدر.

محدوديات استخلاص الجذور. يتمّ استخلاص الجذور إحصائياً عبر حذف الزوائد والمطابقة مع القاموس، دون تصنيف صرفي أو تعليق على أقسام الكلام. لم يُقَس معدل خطأ رسمي. الجذور ذات الحروف الضعيفة (ألف، واو، ياء) والصيغ الرباعية معرّضة بشكل خاص للتعريف الخاطئ. ينبغي التعامل مع نسب الجذور باعتبارها احتمالية لا قطعية.

مشتركات مولّدة بالذكاء الاصطناعي. من بين 1,127 مدخلاً للمشتركات، وُلّد 493 بمساعدة الذكاء الاصطناعي (Claude API) وراجعها بشر يدوياً. على الباحثين في الأعمال الرسمية التحقق من علاقات الاشتراك بشكل مستقل.

{% else %}

The Peshitta as translation. The mainstream scholarly position holds that the Peshitta New Testament is largely a translation from Greek originals, not an independent Aramaic composition. Root analysis therefore reflects the translator’s lexical choices, not necessarily the original author’s vocabulary. The Peshitta Old Testament was translated primarily from Hebrew, though some portions may preserve independent Aramaic traditions.

Targums as interpretive translations. Targum Onkelos is an interpretive Aramaic rendering of the Hebrew Torah. Its vocabulary reflects the targumist’s paraphrase and exegetical expansion, not a verbatim correspondence with the Hebrew source text.

Root extraction limitations. Root extraction is performed statistically via affix stripping and dictionary matching, without morphological tagging or part-of-speech annotation. No formal error rate has been measured. Weak-letter roots (containing ʾalep, waw, yod) and quadriliteral forms are particularly prone to misidentification. Users should treat root attributions as probabilistic, not definitive.

AI-generated cognates. Of the 1,127 cognate entries, 493 were generated with AI assistance (Claude API) and manually reviewed. Researchers conducting formal work should independently verify cognate relationships.

{# ── 8. TECHNICAL NOTES ── #}

{% if lang == 'es' %}Notas técnicas{% elif lang == 'he' %}הערות טכניות{% elif lang == 'ar' %}ملاحظات تقنية{% else %}Technical Notes{% endif %}

{% if lang == 'es' %}

Todos los datos se cargan al inicio desde archivos CSV y JSON locales — no hay dependencias de API en tiempo de ejecución. La aplicación está construida con Flask (Python) y D3.js, con JavaScript vanilla para la interfaz. Las tipografías siríacas se proporcionan mediante la familia Noto Sans Syriac (OFL-1.1) a través de Google Fonts.

Código fuente: github.com/Jossifresben/aramaic-root-atlas

{% elif lang == 'he' %}

כל הנתונים נטענים בעת ההפעלה מקבצי CSV ו-JSON מקומיים — אין תלויות API בזמן ריצה. האפליקציה בנויה עם Flask (Python) ו-D3.js, עם JavaScript ונילה לממשק. גופני הסורית מסופקים באמצעות משפחת Noto Sans Syriac (OFL-1.1) דרך Google Fonts.

קוד מקור: github.com/Jossifresben/aramaic-root-atlas

{% elif lang == 'ar' %}

تُحمَّل جميع البيانات عند بدء التشغيل من ملفات CSV وJSON محلية — لا توجد تبعيات واجهات برمجة تطبيقات أثناء التشغيل. بُنيت التطبيق بـ Flask (Python) وD3.js، مع JavaScript صرف للواجهة. تُوفَّر خطوط السريانية عبر عائلة Noto Sans Syriac (OFL-1.1) من Google Fonts.

الكود المصدري: github.com/Jossifresben/aramaic-root-atlas

{% else %}

All data is loaded at startup from local CSV and JSON files — there are no runtime API dependencies. The application is built with Flask (Python) and D3.js, with vanilla JavaScript for the frontend. Syriac fonts are provided by the Noto Sans Syriac family (OFL-1.1) via Google Fonts.

Source code: github.com/Jossifresben/aramaic-root-atlas
